# Node & Topology Checker (ปลั๊กอิน QGIS)

ปลั๊กอินตรวจสอบคุณภาพข้อมูลรังวัด/แปลงที่ดิน สำหรับ QGIS 3.28–3.40 LTR (PyQt5)

ตรวจสอบได้ 3 อย่าง โดยเน้นแก้ปัญหา **การรายงานทับซ้อนผิดพลาด (false positive)** ของ
Topology Checker เดิม ที่ยังฟ้องว่าแปลงซ้อนกันทั้งที่แก้ให้แนบสนิทแล้ว

## ความสามารถ

1. **ตรวจการทับซ้อน (Overlap)** ระหว่างโพลิกอน — แสดงพื้นที่ที่ซ้อนเป็นสีแดง
2. **ตรวจช่องว่าง (Gap)** ที่ถูกโพลิกอนล้อมรอบ — แสดงช่องว่างเป็นสีแดง
3. **ตรวจ Node/Vertex** ของ POLYGON ที่ไม่มีหมุด POINT ตรงกัน — แสดงจุดสีแดง + รายการ
4. เลือกชั้นข้อมูล POINT / POLYGON แยกกัน และเลือกขอบเขต **ตรวจทั้งหมด** หรือ **เฉพาะหน้าต่างปัจจุบัน**
5. ตารางผลลัพธ์ — **ดับเบิลคลิกที่แถวเพื่อซูม** ไปยังตำแหน่งที่ผิดพลาด (มีไฮไลต์กะพริบ)
6. ปุ่ม **ตรวจสอบอัปเดตปลั๊กอิน** เทียบเวอร์ชันกับ GitHub และอัปเดตได้ในคลิกเดียว

## จุดเด่น: ทำไมไม่ฟ้อง Overlap ผิดพลาดเหมือนเดิม

ปลั๊กอินนี้ใช้ **ค่าความคลาดเคลื่อน (Tolerance)** ร่วมกับเทคนิค:
- ซ่อม geometry ให้ถูกต้องก่อน (`makeValid`) — กัน false positive จากเส้นตัดตัวเอง
- นับเฉพาะ *ส่วนที่เป็นพื้นที่* ของการตัดกัน — ขอบที่แนบกัน (เส้น) ไม่ถือเป็นการทับซ้อน
- ใช้ *negative buffer* กรองเศษ sliver ที่บางกว่า Tolerance — เศษจาก snapping/floating point
  จะไม่ถูกรายงาน

> พิสูจน์ด้วยชุดทดสอบ (`tests/run_checks_test.py`): กรณีแปลงแนบขอบสนิท และกรณี sliver บางกว่า
> Tolerance จะรายงาน 0 รายการ ขณะที่การซ้อนจริงยังตรวจพบครบ

## การติดตั้ง

1. คัดลอกโฟลเดอร์ `QGIS_Node_TopologyChecker` ทั้งโฟลเดอร์ไปไว้ที่
   `%APPDATA%\QGIS\QGIS3\profiles\default\python\plugins\`
2. เปิด QGIS → เมนู `ปลั๊กอิน` → `จัดการและติดตั้งปลั๊กอิน` → แท็บ `ติดตั้งแล้ว` → ติ๊กเปิดใช้งาน
   **Node & Topology Checker**
3. จะมีไอคอนบนแถบเครื่องมือ และเมนู `Vector ▸ Node & Topology Checker`
   (วางไฟล์ไอคอนไว้ที่ `resources/icon.png` ตามคำแนะนำใน `resources/README.txt`)

## วิธีใช้งาน

1. เปิดหน้าต่างปลั๊กอิน (คลิกไอคอน) — หน้าต่างจะเทียบชิดขวา
2. เลือก **ชั้นข้อมูล POLYGON** (และ **POINT** หากจะตรวจ Node)
3. ตั้ง **ค่าความคลาดเคลื่อน (Tolerance)** ให้เหมาะกับข้อมูล (หน่วยเดียวกับ CRS เช่น เมตร)
4. เลือกขอบเขต **ตรวจทั้งหมด** หรือ **เฉพาะหน้าต่างปัจจุบัน**
5. ติ๊กรายการที่ต้องการตรวจ แล้วกด **▶ ตรวจสอบ Topology**
6. ผลลัพธ์แสดงเป็นชั้นสีแดง + ตาราง → ดับเบิลคลิกแถวเพื่อซูมไปแก้ไข
7. กด **ล้างผลลัพธ์** เพื่อลบชั้นผลลัพธ์และเคลียร์ตาราง

## ตั้งค่าระบบอัปเดต (GitHub)

1. นำโฟลเดอร์ปลั๊กอินขึ้น GitHub (โครงสร้างไฟล์ต้องอยู่ที่ราก repo และมี `metadata.txt`)
2. ในหน้าต่างปลั๊กอิน กด **ตั้งค่า GitHub** แล้วกรอก `owner/repository`
   (เช่น `phakphum/QGIS_Node_TopologyChecker`)
3. เวลาปล่อยเวอร์ชันใหม่: แก้ไฟล์ในโปรเจกต์ + **เพิ่มเลข `version=` ใน `metadata.txt`** แล้ว push ขึ้น branch หลัก
4. ผู้ใช้กด **ตรวจสอบอัปเดตปลั๊กอิน**:
   - ถ้ามีเวอร์ชันใหม่ → แจ้งเวอร์ชัน + ปุ่ม *อัปเดตเลย* (ดาวน์โหลด+ติดตั้งทับ แล้วให้ปิด/เปิด QGIS ใหม่)
   - ถ้าล่าสุดแล้ว → แจ้ง **"ใช้เวอร์ชันล่าสุดแล้ว"**

> หมายเหตุ: ตั้งค่าเริ่มต้น `DEFAULT_REPO` ใน `update_checker.py` ได้ (ปัจจุบันเป็น `CHANGE_ME/...`)
> การเทียบเวอร์ชันใช้ค่า `version=` ใน `metadata.txt` จึงต้องเพิ่มเลขเวอร์ชันทุกครั้งที่ปล่อยของใหม่

## โครงสร้างไฟล์

| ไฟล์ | หน้าที่ |
|------|--------|
| `plugin.py` | จุดเชื่อมกับ QGIS (เมนู/ทูลบาร์) เปิดหน้าต่าง |
| `topology_dock.py` | หน้าต่าง UI ภาษาไทย + เชื่อมทุกฟีเจอร์ |
| `checks/geometry_utils.py` | ฟังก์ชันแก้ false positive (makeValid, polygonal_part, is_significant) |
| `checks/overlap_check.py` | ตรวจการทับซ้อน |
| `checks/gap_check.py` | ตรวจช่องว่าง |
| `checks/node_check.py` | ตรวจ Node/Vertex กับ POINT |
| `result_layers.py` | สร้างชั้นผลลัพธ์สีแดง |
| `check_task.py` | รันการตรวจแบบ background (ไม่ให้ QGIS ค้าง) |
| `update_checker.py` | เทียบเวอร์ชัน + ดาวน์โหลด/ติดตั้งจาก GitHub |
